Github Notları

Temel Terimler

  1. repository : Kısa ismiyle repo, kodlarımızın saklandığı yerdir
  2. master : Depodaki ana proje sürümüdür
  3. branch : Git ağaç mantığı ile çalışır ve projedeki geliştirmelerimizi ağaç dalı mantığı ile ayrı dallarda saklayabilir, sonrasında master branch de birleştirebiliriz
  4. merge : branch lerde yapılan işlemlerin master da birleştirilmesidir.
  5. commit : Commit işlemi lokaldeki dosyalarımızı depoya gönderilmek üzere hazırlar
  6. push : Dosyalarımızı depoya göndermemizi sağlar
  7. pull : Depodaki dosyalarımızı çekmemizi sağlar
  8. .gitignore : Gİt in takip etmesini istemediğpimiz dosya ve klasörleri belirttiğimiz dosyadır.

Vs code da projede bir değişiklik yapıldıktan sonra bunu hithub a göndermek için source control tabını açtık. sonra Message alanına yapılan değişiklikle alakalı bir mesaj yazdık. Son olarak Commit butonuna basıp depoya gönderilmek üzere hazırladık ve Sync butonuna basarak Push işlemini yaparak değişikliği github a yolladık.

Temel Git Komutları

  1. git add veya git add * , gitt add index.html : Klasöre eklenen dosyaları gönderilmek üzere sahneye alma (Stage)
  2. git status : Klasöründeki tüm değişiklikleri gösterir.
  3. git commit -m "ilk commit" : Depoya gönderilmek için commit işlemi yapılır
  4. git commit -a -m "Mesajımız" : Staging e ekle ve commitle
  5. git push : değişikleri depoya gönderme komutu
  6. git pull : uzak depodaki dosyaları lokale indirme

Github a yüklediğimiz projenin gizlilik ayarını githubdaki proje içerisine girip > settings den ayarları açıp en altta ki gizlilik ayarlarından yapabiliriz.

Github dan projeyi silmek için yine settings den danger zone alanındaki DELETE THİS REPOSİTORY butonuna basrak gelen pencereden bizden istediği yazıyı kutucuğua girerek silmeyi onaylayıp repoyu silebiliriz.

Bir projeyi bir ekiple geliştirmek istersek setting deki sol menüden collaborators sekmesinden giriş yaparak açılan ekrandan diğer github kullanıcılarını projeye dahil ederek ortak geliştirme yapabiliriz.
